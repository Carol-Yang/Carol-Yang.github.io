Git 學習筆記 (四) - 狀況題

發表於 2019-08-29 分類於 Git

八、狀況題

1. 在 Git 內加入 “空資料夾”

因 Git 依據 “檔案內容” 計算，故空資料夾無法提交，可在空資料夾加入 .keep 或 .gitkeep 檔案即可 (慣例上)

2. 忽略特定檔案不做 Git

  • 在資料夾中加入 .gitignore (檔案要在未追蹤狀態，否則還是會被 commit)

3. 誤刪檔案 (ex: 使用 $ git rm )

4. 救回使用 reset - -hard 的 commit

5. 臨時要做別的事

5-1. 方法一： 使用 Reset

5-2. 方法二： 使用 stash


$ git stash list

6. 刪除 Git 中 的檔案

6-1. 方法一： 把 .git 目錄刪除，再重新 commit

6-2. 方法二：使用 filter-branch 指令 ( 可一次大量修改 commit )

6-3. 復原 filter-barch

6-4. 如何完全清除檔案

在使用 rebase 或 filter-branch 後，檔案還是在 (以下以 filter-branch 為例)

1. .git/refs/original/refs/heads/master
2. reflog
3. unreachable 懸浮物件
