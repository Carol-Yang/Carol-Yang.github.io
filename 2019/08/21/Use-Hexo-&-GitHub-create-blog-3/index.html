利用 Hexo + GitHub 建立部落格 (三) - Hexo _config.yml 配置指南 2

發表於 2019-08-21 更新於 2019-08-29 分類於 Hexo

昨天講解完了 Hexo 的 _config.yml 配置文件，今天來另外說明其中幾個配置~
這些設定在一剛開始建立 Hexo blog 時，都是非必要的選項，可以跳過。

重點是建立部落格後要持續寫文章!!!!

一、文章連結設置

permalink 可以設置文章的永久連結，以下是設置的一些範例

permalink 設置 顯示
:year/:month/:day/:title/ https://carolblog.github.io/2018/08/21/hello-world
:year-:month-:day-:title.html https://carolblog.github.io/2018-08-21-hello-world.html
:category/:title https://carolblog.github.io/begin/hello-world

變數

permalink 中可使用預設的變數有

變數 描述
:year 文章的發表年份（4 位數）
:month 文章的發表月份（2 位數）
:i_month 文章的發表月份（去掉開頭的 0）
:day 文章的發表日期 (2 位數)
:i_day 文章的發表日期（去掉開頭的 0）
:title 文章名稱
:id 文章 ID
:category 分類

當文章標題為中文時

當一篇名為 “利用 Hexo + GitHub 建立部落格 (二)” 的文章被建立時，
以我們在 permalink: :year/:month/:day/:title/ 中的設定，文章網址應該是

https://carol-yang.github.io/2019/08/21/利用 Hexo + GitHub 建立部落格 (二)/

但是因為中文會被轉譯為十六進制編碼，因此實際網址會是 ( 請把網址複製到記事本試試 )

https://carol-yang.github.io/2019/08/21/%E5%88%A9%E7%94%A8-Hexo-GitHub-%E5%BB%BA%E7%AB%8B%E9%83%A8%E8%90%BD%E6%A0%BC-%E4%BA%8C/

除了不好看外，也會影響 SEO，這時候 permalink 與 permalink_defaults 設定就派上用場了

1. 自訂義變數

先自訂一個 urlname 作為我們放英文標題的變數，並在 permalink_defaults 內設置它的默認值

2. 在文章中加入變數

在每篇文章的 Front-matter 區塊中，加入變數

＊ 注意
在 urlname 的內容，不要使用空白鍵及特殊符號，也會與中文一樣轉成代碼
每篇文章都要在 Front-matter 中加入 urlname，否則未寫 urlname 的文章，網址中的 urlname 區塊會顯示 undefined
例如：
https://carol-yang.github.io/2019/08/21/undefined/

二、分類與標籤設置

除了文章內頁網址外，分類與標籤也同樣會因為中文轉為十六進制編碼的關係，造成同樣的問題

在開始做之前，我們先來說說 default_category，default_category 的意思是當你這篇文章沒有設定分類，
它就會自動把這篇文章加入到 uncategorized 這個分類中，當然 uncategorized 是可以改名稱的。

例如：
當你在 permalink 加入了分類變數，想要文章內頁網址也包含分類名
permalink: :year/:month/:day/:category/:urlname/
但如果你某篇文章並沒有自己設定分類呢？
它就會放在默認分類中，預設的分類名 uncategorized
所以文章網址就會呈現：
https://carol-yang.github.io/2019/08/21/uncategorized/Use-Hexo-&-GitHub-create-a-blog-2/

開始設定標籤及分類的英文名

現在我們開始，把中文的標籤及分類，更改成顯示是中文，網址是顯示英文吧!

以分類為例：
設一個分類名為 “30 天 blog 挑戰”
當進入”30 天 blog 挑戰”分類頁面，因為中文的關係，網頁會呈現
https://carol-yang.github.io/categories/30-%E5%A4%A9-blog-%E6%8C%91%E6%88%B0/

在 category_map 設定中，加入 中文分類名: 英文分類名

設定好後，網址就會呈現
https://carol-yang.github.io/categories/30-days-challenge/

標籤頁面也是同樣做法，修改 tag_map

在”開始 Hello World”的標籤頁面網址就會呈現：
https://carol-yang.github.io/tags/begin/

當分類與標籤原本就是英文時，就不需再另外設置了，因為原本就是英文嘛 ><
