利用 Hexo + GitHub 建立部落格 (四) - 我的 Hexo _config.yml 設定

發表於 2019-08-22 更新於 2019-08-29 分類於 Hexo

說完了在 _config.yml 的配置介紹，今天來說說我的 _config.yml 配置，

主題也會有另外的 _config.yml 需要設置，改天再另外說明

以下內容並未把全部都貼上，只貼上有更改的

網站介紹

中文語言可使用 language: zh-tw， 而在這裡我設置 language: zh-TW 是依照 next 主題 所設定，
語言可在 themes/next/languages/ 資料夾內查找

注意：
語言設置的大小寫很重要

網站網址

因為我是使用 GitHub 來放置我的 blog，所以在網站網址先寫入 http://帳號名稱.github.io

關於 permalink 及 permalink_defaults: 設定，詳細可看 利用 Hexo + GitHub 建立部落格 (三)

寫作

在寫作方面

  • 使用 :year-:month-:day-:title.md 來做文章檔名，例如：
    2019-08-19-利用-Hexo-GitHub-建立部落格-一.md
  • 設置 post_asset_folder: true ，在創建文章的同時，建立一個與文章同名的資料夾，這個資料夾可用來放置這篇文章的圖片與相關資料

部屬

在 _config.yml 裡，部屬設定非常重要!

repository 及 branch 需自行寫上，預設並沒有

  • 使用 GitHub 來建立 blog，所以 type: git
  • 在 GitHub 中創建 repo 時會給儲存庫網址，而我是使用 SSH keys，所以寫入
    repository: git@github.com:carolblog/carolblog.github.io.git

  • 分支上，設定 branch: master ，自訂名稱也可，但我習慣使用預設名稱

在設定好 Hexo 的 config.yml 檔案後，終於要開始說與寫作相關的東西了 ~
